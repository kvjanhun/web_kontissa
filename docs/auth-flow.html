Authentication & Data Flow

A Sequence Diagram showing how the Nuxt SPA frontend interacts with the Flask API on admin login for authentication and protected state management using Pinia and HttpOnly cookies.

sequenceDiagram autonumber actor User participant Nuxt as Nuxt SPA (Pinia) participant Flask as Flask API participant DB as SQLite Note over User,DB: 1. Initial Login User->>Nuxt: Enters credentials at /login Nuxt->>Flask: POST /api/login {email, password} Flask->>DB: Query User by email DB-->>Flask: User record (scrypt hash) Flask->>Flask: Validate password hash Flask-->>Nuxt: 200 OK + Set-Cookie: session (HttpOnly) Nuxt->>Nuxt: Update AuthStore (isAuthenticated: true) Nuxt-->>User: Redirect to /admin Note over User,DB: 2. Accessing Protected Data User->>Nuxt: Navigates to /admin/sections Nuxt->>Nuxt: Vue Router Middleware allows navigation Nuxt->>Flask: GET /api/sections (Browser sends session cookie) Flask->>Flask: @admin_required validates session Flask->>DB: Fetch sections DB-->>Flask: Rows Flask-->>Nuxt: 200 OK (JSON Data) Nuxt->>Nuxt: Render Vue Components Nuxt-->>User: Displays Admin Dashboard Note over User,DB: 3. Session Persistence (Page Reload) User->>Nuxt: Hard Refresh /admin Nuxt->>Flask: GET /api/me (on app mount) Flask-->>Nuxt: 200 OK (User data) + session cookie validated Nuxt->>Nuxt: Restore AuthStore state Nuxt-->>User: UI remains logged in
