Sanakenno Game Flow

A Sequence Diagram illustrating the zero-trust architecture of the Finnish Spelling Bee clone, showing how client-side hashing protects the word list and how state persists.

sequenceDiagram autonumber actor Player participant Nuxt as Nuxt SPA (Client) participant Local as localStorage participant Flask as Flask API participant DB as SQLite Note over Player,DB: 1. Game Initialization Player->>Nuxt: Navigates to /sanakenno Nuxt->>Flask: GET /api/kenno Flask->>Flask: Calculate today's puzzle slot Flask->>DB: Fetch letters & center Flask->>Flask: Compute SHA-256 hashes of valid words Flask-->>Nuxt: 200 OK (letters, center, word_hashes, hint_data) Nuxt->>Local: Check saved state (sanakenno_state) alt State is from today's puzzle Local-->>Nuxt: Restore found words & score else State is old Local-->>Nuxt: Null (Reset state) end Nuxt-->>Player: Render Honeycomb UI & Current Score Note over Player,DB: 2. Gameplay Loop (Zero-Trust Validation) Player->>Nuxt: Types word and presses Enter Nuxt->>Nuxt: Compute SHA-256 hash of input word Nuxt->>Nuxt: Compare input hash against word_hashes array alt Hash Matches (Valid Word) Nuxt->>Nuxt: Calculate word score (length + pangram bonus) Nuxt->>Local: Save updated found words & score Nuxt-->>Player: Update UI (Score increases, word added to list) else Hash Not Found (Invalid Word) Nuxt-->>Player: Shake animation (Invalid word) end Note over Player,DB: 3. Progression & Achievements Nuxt->>Nuxt: Check if new score crosses a Rank Threshold alt Threshold Crossed Nuxt-->>Player: Show Rank Up Notification / Celebration Nuxt->>Flask: POST /api/kenno/achievement (Async, fire-and-forget) Flask->>Flask: Deduplicate via session cookie Flask->>DB: Save anonymous achievement stat Flask-->>Nuxt: 201 Created (or 200 Already Recorded) end
